## 案例 001 —「合規」：灰色地帶的精準拿捏 ⭐

- **類型**：灰色地帶（不是硬錯）
- **原句**：「正式成為 C2PA **合規**產品」
- **支語點**：合規
- **台灣版**：符合 C2PA **規範**的產品 ／ 符合標準
- **為什麼是「灰色」不是「硬錯」**：
  台灣金融、法律圈本來就用「合規」(compliance)，所以不能一刀切說它是支語；
  但「**合規產品**」這個組合讀起來偏陸味，台灣更自然的講法是「符合規範」。
  關鍵在**組合與語境**，不是這個詞本身。
- **教學點 / 發文角度**：
  支語不是非黑即白。最考驗判斷的，不是「視頻、質量、默認」這種一眼就知道的**硬錯**，
  而是「合規」這種——**台灣也用、但放進特定組合裡就走味**的灰色詞。
  好的支語警察要會分「硬錯（直接改）」和「灰色（交給你決定）」，
  而不是看到疑似就無腦替換。**這種分寸感，才是真正有用的地方。**
- **出處**：一篇 AI 議題貼文實測（2026-06）
- **同篇正確放行（沒誤抓）**：浮水印（✅ 台灣正解，中國叫「水印」）、數位（✅ 中國叫「數字」）、
  截圖 / 像素 / 溯源 / 自媒體（✅ 台灣通用）—— 示範了「寧可漏抓、不要錯抓」。

## 案例 002 —「土豆」：同名異實假朋友，硬改會改錯意思 ⭐⭐

- **類型**：同名異實（false friend）→ 字典的第 4 級 `context`，**絕不自動改**
- **真相**：
  - 🇹🇼 台灣：**土豆 = 花生**（源自台語，講習慣了把花生叫土豆）；**馬鈴薯 = 馬鈴薯**（只有這個講法）
  - 🇨🇳 中國：**土豆 = 馬鈴薯**
- **為什麼這是最危險的一類**：
  「土豆」對台灣人**根本沒錯**（就是花生）。如果工具把它當支語、自動改成「馬鈴薯」，
  反而把對的字改錯——這比漏抓還糟。同理「質量」（台=物理 mass／陸=quality）、
  「水平」（台=horizontal／陸=程度）都是同字不同義。
- **正確處理**：不歸 hard、不歸 grey，獨立成 **`context` 級**：偵測到就跳出
  「⚠️ 同名異實：看你是哪個意思」的提示，把兩岸語義講清楚，**讓人自己判斷、程式不動手**。
- **教學點 / 發文角度**：
  支語警察最難、也最該謹慎的不是「抓到支語」，而是「**分清哪些是真支語、哪些是台灣本來就有的字**」。
  「土豆」就是最好的例子——同一個詞，台灣是花生、中國是馬鈴薯。好的工具要懂這個差異，
  而不是看到字一樣就開槍。**寧可提醒，不要誤殺。**
- **來源**：使用者親自指正（2026-06）。研究 agent 也警告過「同名異實」整類不可當替換詞 ingest。

## 案例 003 —「奧巴馬 / 悉尼 / 奔馳」：外國名的兩岸音譯差異 ⭐⭐

- **類型**：譯名差異（同一個外國人名/地名/品牌，兩岸音譯不同）
- **對照範例**：

  | 領域 | 🇨🇳 中國譯名 | 🇹🇼 台灣譯名 |
  |---|---|---|
  | 人名 | 奧巴馬 / 特朗普 / 普京 / 喬布斯 | 歐巴馬 / 川普 / 普丁 / 賈伯斯 |
  | 地名 | 悉尼 / 新西蘭 / 戛納 / 硅谷 | 雪梨 / 紐西蘭 / 坎城 / 矽谷 |
  | 品牌 | 奔馳 / 大眾汽車 / 雷克薩斯 | 賓士 / 福斯汽車 / 凌志 |

- **為什麼這類最隱形、最該收**：
  視頻、質量這種詞，台灣人一眼就覺得「怪怪的」；但**譯名不一樣**——你寫「悉尼」「奧巴馬」「奔馳」，
  字面上沒有簡體、沒有明顯陸味，很多人不會察覺那是中國譯法，**這是最容易在不知不覺中流出去的支語**。
  尤其用 AI 寫稿、AI 引用外電時，幾乎都吐中國譯名（因為語料來源）。
- **教學點 / 發文角度**：
  支語不只是「視頻 vs 影片」這種詞彙，**連外國人的名字、城市、車廠的叫法都有兩套**。
  「奔馳 vs 賓士」就是最好的例子——同一台車，叫法洩漏你讀的是哪邊的內容。
  支語警察會把這類譯名差異也抓出來，讓你的稿從詞彙到專有名詞都是台灣的。
- **資料來源說明**：NAER 官方「外國地名/人名譯名」是英→台單向（沒有中國譯名欄），無法直接成對；
  所以這批用**精選常見差異** + OpenCC 既有譯名對照（已涵蓋奧巴馬/悉尼等）。放普通版，常態就抓。
- **來源**：使用者指定「奔馳→賓士」為代表範例（2026-06）。

## 案例 004 —「互聯網 → 互聯 / 聯網」：子字串重複命中（ULTRA 過度法辦）⭐

- **類型**：工具行為特例 / 子字串重疊（主要出現在 L3 支語檢察官模式）
- **現象**：一句「互聯網上轉發超多」，在檢察官模式被報了三次：
  - `互聯網 → 網際網路`（hard，正確）✅
  - `互聯 → 互連`（grey，電力學學術詞）← 是「互聯網」的前兩字
  - `聯網 → 網路連結`（grey，計算機學術詞）← 是「互聯網」的後兩字
  → **同一個詞被拆成三段、各自命中一次**。
- **為什麼會這樣**：偵測是「子字串比對」。普通版只有「互聯網」這種完整常用詞 → 乾淨；
  但 L3 展開了上萬個**短學術片段**（互聯、聯網…），長複合詞就會「內含」好幾個短片段，全部觸發。
- **為什麼主要在 L3 出現**：互聯/聯網 這類細碎學術片段只在學術層（進階/檢察官才載入）。
  *但要精確說*：子字串重疊在任何層級理論上都可能發生（普通版若同時有長短詞也會），
  通用根治法是**最長匹配抑制**——已實作於 `police.py correct_hard`（位置式、長詞先佔位、
  重疊短詞跳過），所以現在「互聯網」只報一次、不再裂成互聯/聯網。分級制讓常態普通版更乾淨。
- **教學點 / 發文角度**：
  「**詞庫越大不一定越好**」的活教材。當支語警察把標準開到最高，連「互聯網」都能挑剔成三條，
  這就是迷因裡笑的「矯枉過正支語警察」——只是這次是工具層面的副作用。
  好的設計是**知道什麼時候該收手**：所以常態用普通版，最高階只在發文前把關一次、且結果要人複核。
- **可改進方向**（尚未實作）：「最長匹配優先」——若某個短支語的命中範圍**完全落在**另一個已命中的
  長支語詞範圍內，就抑制那個短的（互聯網已命中 → 不再報互聯/聯網）。要做我可以加進 `police.py`。
- **來源**：使用者在檢察官實測時發現（2026-06）。
